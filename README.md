# LLM-Powered Market Trend Discovery from Airline Twitter Data
### Overview

This project implements an end-to-end pipeline for discovering customer complaint trends from airline Twitter comments using embeddings, clustering, and Large Language Models (LLMs).

The system transforms large volumes of unstructured social media feedback into structured insights that help identify recurring customer issues, emerging complaint patterns, and potential operational problems.

The pipeline combines semantic embeddings, density-based clustering, and LLM-based analysis to automatically group similar comments and extract meaningful information that can support data-driven business decisions.

## Project Architecture

The project is divided into multiple stages, each responsible for a different part of the analysis pipeline.

Raw Twitter Data > Data Cleaning & Feature Engineering > Embedding Generation >(OpenAI) > Semantic Clustering (HDBSCAN) > LLM Cluster Analysis > Trend Analysis & Visualization

### Part 1: Data Cleaning and Feature Engineering
**Objective:** Prepare airline Twitter comments and complementary dataset features for embedding generation and trend analysis.

**Description:** In this stage, the raw Twitter comments are cleaned and normalized to generate high-quality text inputs for embeddings. The dataset also includes additional numerical and categorical features that support the trend analysis. These variables are processed, standardized, and encoded to ensure consistency and compatibility with downstream machine learning workflows. This step ensures the dataset is reliable and ready for semantic analysis.

### Part 2: Embedding Generation with OpenAI
**Objective:** Generate semantic embeddings from airline Twitter comments using an OpenAI embedding model.

**Description:** The cleaned text data is converted into vector embeddings using an OpenAI model optimized for semantic similarity tasks. To optimize API usage and reduce costs, the pipeline implements an embedding caching mechanism:
- If embeddings have already been generated, the system loads them from a stored file.
- If embeddings are not found, they are generated through the OpenAI API and saved for future reuse.

This approach prevents unnecessary API calls and helps preserve OpenAI API credits.

### Part 3: Semantic Clustering with HDBSCAN
**Objective:** Identify groups of semantically similar Twitter comments using embedding-based clustering.

**Description:** Once embeddings are generated, the HDBSCAN clustering algorithm is used to detect dense regions in the embedding space. This allows the system to automatically group comments discussing similar topics or issues. HDBSCAN is particularly useful because:
- It does not require specifying the number of clusters
- It can detect outliers naturally

Each cluster represents a set of related customer comments that will be analyzed in the next stage.

### Part 4: LLM-Based Cluster Analysis
**Objective:** Use a Large Language Model to analyze clusters of related comments and extract structured insights.

**Description: **A custom prompt is designed to send the comments from each cluster to an LLM for analysis. The model returns a structured JSON output containing relevant information about the cluster. The JSON output is then parsed and merged into the main dataset, enriching the data with structured insights.

### Part 5: Trend Analysis and Insight Generation
**Objective:** Analyze the structured outputs generated by the LLM to identify patterns and emerging customer complaint trends.

**Description:** The enriched dataset is used to perform exploratory analysis on the information extracted by the LLM.

Various visualizations are generated to analyze relationships between complaint topics, severity levels, trend signals, customer issues. These analyses help identify recurring problems and emerging patterns in airline customer feedback.

The resulting insights can support data-driven decision making, helping organizations monitor customer issues and detect potential operational problems early.
